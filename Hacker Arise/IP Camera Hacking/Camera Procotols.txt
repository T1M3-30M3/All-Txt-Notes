Common Used Protocols in CCTV IP Cameras

📡 Streaming & Media Protocols:
RTSP (Real-Time Streaming Protocol) → Standard for video/audio streaming from IP cameras.
RTP/RTCP (Real-Time Protocol / Real-Time Control Protocol) → Works with RTSP to deliver media streams.
HTTP(S) → Used for accessing live video via web browsers, API calls, or cloud services.
ONVIF (Open Network Video Interface Forum) → An industry standard that ensures interoperability between cameras and NVRs from different vendors.

---

⚙️ Control & Device Management Protocols:
SNMP (Simple Network Management Protocol) → For monitoring camera health/status.
DHCP → Automatically assigns IP addresses to cameras.
DNS → For resolving camera names when connecting remotely.
UPnP (Universal Plug and Play) → Helps discover cameras automatically on local networks.

---

🔒 Security Protocols:
HTTPS (SSL/TLS) → Encrypts web interface and video streams.
802.1X → Used in enterprise setups for secure network access.
SRTP (Secure RTP) → Encrypts audio/video streams for extra security.

---

🛠️ Other Protocols Sometimes Used:
FTP / SMTP → For sending snapshots or alerts (email, storage).
NTP (Network Time Protocol) → Keeps camera clocks synchronized.
MQTT / WebSockets → Some smart CCTV systems use these for IoT integrations.

---

👉 In short:
RTSP/RTP → video streaming
ONVIF → interoperability
HTTP/HTTPS → web & API access
SNMP/DHCP/NTP → management & services
