XSS: Mitigation:

* Input sanitization
* Escaping: encoding all the input that the user ends
* Strong input validation for limited input with sanitization
* WAF
* Filter input on arrival: At the point where the user is received, filter as strongly as possible based on what is expected or voiced input.
* Encode data on output : At the point where user-controllable data is output in http responses,encode the oytput to prevent it rom being interpreted as active content,Depending on the output context, this might require applying combinations of HTML, URL, JS, and CSS encoding
* Use appropritte response headers - To prevent XSS in HTTP response that arenot intended in contain any html or JS, you can use the Content-Type and X-content-Type-Options headers to ensure that browsers interpret the responses in the way you intend
* Content Security Policy - As a last line of defense,you can use Content Security Policy(CSP) to reduce the severity of any XSS vulnerabilities that still occur

Html Injection:

Html stands for Hypertext Markup Language. HTML is the standard markup language for web pages. The collection of such web pages makes a website. HTML elements are the building blocks of HTML pages. HTML elements are represented by <> tags. Each tag has a different working.

Attack :

Html Injection is a vulnerability that occurs in web applications that allows users to insert an HTML vode via a specific parameter, for example, an entry point.

Exploited with: Social engineering in order to trick valid users into the application, open websites, or insert their credentials in a fake login. From there, it will redirect the users to a page that captures cookies and credentials.
