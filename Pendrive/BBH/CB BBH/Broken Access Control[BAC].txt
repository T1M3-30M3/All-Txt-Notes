BAC is a type of application security vulnerability that enables users to access data and functionalities that they should not have access to.

IDOR: 

Tools to Use:
▶ Postman: For testing and documenting API requests
▶ Burp Suite: To intercept and manipulate API traffic

Where To Find?
▶ Look for admin actions that might be accessible to regular users
▶ Endpoints with actions like data modification or deletion
▶ Endpoints that return information specific to a user

How to Exploit?
▶ Change user roles in JWT and attempt to access restricted functions
▶ Modify resource IDs in API requests to access data of other users
▶ Use lower privilege tokens to access higher privilege endpoints

Common Types of IDOR:

--> GET-based IDOR (URL Manipulation):
▶ Numaric IDOR
▶ username IDOR
▶ Resource Path Manipulation

--> POST-based IDOR (Form and Parameter Manipulation)
--> Cookie-based IDOR
--> API-based IDOR
--> Database-based IDOR
--> Session-based IDOR
--> Encodes ID: hashes ID, encoded ID

▶ Url, API, Ajax Call, Understand Hashing/Encoding, User Roles
