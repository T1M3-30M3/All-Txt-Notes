LFI:

▶ LFI stands for Local file Inclusion
▶ It is a file inclusion vulnerability that is usually used to read sensitive files
▶ Attacker manipulates input fields or parameters that are used by the applications to includes files from the local file system
▶ Access sensitive files such as /etc/passwd, /etc/shadow, /home/usr/.ssh/id_rsa

LFI vs Path Traversal:

LFI: LFI specifically targets web applications that dynamically include files based on user supplied input, such as file paths or parameters. Attackers exploit this vulnerability by manipulating these inputs to access files on the server that should not be accessible.The goal of LFI is to include and execute files that reside locally on the server where the web application is hosted

Path Traversal: It is vulnerability that allows an ttacker to access files or directories that are outside the web applications root directory.It invloves manipulating file path inputs to navigate to directories and retrieve files that should be retricted.The objective of path traversal is to traverse beyond the intended directory structure, potentially accessing sensitive system files or directories

* LFI Not always RFI:
▶ LFI May not necessarily be an RFI. This is primarily because of Three Reasons:
▶ The vulnerable function may not allow including remote URLS
▶ You may only control a portion of the filename and not the entire protocol wrapper (example: http://, ftp://, https://)
▶ The configuration may prevent RFI altogether, as most modern web servers disable including remot file by default
