JWT Headers:
	{"typ" : "JWT", "alg":"HS256"}

"typ": "JWT" >> Type
"alg":"HS256" >> algorithm

JWT Payload:
	{"iat":"275254254", "iss":"localhost","exp":"757852727","userid":"mountain","isAdmin":"false"}

"iat":"275254254" >> Issued At
"iss":"localhost" >> Issuer
"exp":"757852727" >> Expiry
"userid":"mountain" >> User Identification
"isAdmin":"false" >> User Identification

JWT Signature: header, paylaod, kwy, algorithm

Common Algorithms : HS256, RS256

HS256 Algorithm:
--> HS256 uses SHA256-HMAC(Hash-Based Message Authentication Code)
--> This means SHA256 hash is genarated but by using a secret key
--> So HS256 is SHA256 hash of header, Payload and a Secret => A digital Signature


RS256:
--> RS156 uses RSA and SHA256 algorithm
--> First, SHA256 hash of header and payload is genarated
--> This hash is signed using RSA priate key
--> so RS356 Signature is a signed SHA256 hash of headers, paylaods => A digital Signature
--> This is appended to the Header and payload as 3rd part of JWN tojen
--> One can repuduce this token only if he/she hash private key and usterceot the hsah
