When a browser sends an HTTP request from one origin to another, any cookies, including authentication session cookies, related to the other domain are also sent as part of the request. Many websites interact with subdomains or third-party sites in a way that requires full cross-origin access.

Web Cache Poisoning:

Cache: Caches save copies of responses to reduce load on the backend system. When a cache receives an HTTP request, it looks into the web cache server. If the request is already stored in the cache memory, it will provide you with a response directly without passing the request to the server.

Web Cache Poisoning : It is a technique by which an attacker tricks a web server into storing harmful or incorrect information in its cache memory. In web cache poisoning, the attacker tries to make the server store bad data, so when someone visits the website, they are served that bad data.


Possible Attacks:

* Content Spoofing: An attacker can manipulate the cached content to display incorrect or malicious information to website visitors.
* Session hijacking: By poisoning the cache, an attacker can gain unaythorized access to user sessions. They can tamper with session tokens or cookies stored in the cache, allowing them to impersonate legitimate users and perform malicious actions on their behalf.
* XSS: Web cache poising can be used to inject malicious scripts into the cache, which are then served to multiple users visiting the website.

Broken Access Control: 

What is access control:

* Access control is a security technique that regulates who or what can view or use resources in a computing enviroment.
* in the context of web application,access control is dependent on authentication and session maangement:
	* Authentication identifies the user and confirms that are who they say they are.
	* Session management identifies which subsequent HTTP requests are being made by that same user
	* Access control determmines whether the user is allowed to carry out the action that they are attempting to perform.

* Broken access control is a type of vulnerability that allows unauthorized user to gain access to sensitive data or system.
* Broken access controls are a commonly encountered and critical security vulnerability.

Example of BAC:
* Insecure ID's(IDOR)
* Directory Travesal
* Unauthenticated Endpoint

Vertical Privilege Escalation:

* Vertical access controls are mechanisms that restrict access to sensitive functionality that is not available to other types of users.
* Vertical privilege escalation occurs when an attacker gains access directly to an account with the intent to perform actions as that person.
* The goal here is to access an account to further spread an attack or access data the user has permission to.

Horizontal Privilege Escalation:

* Horizontal access controls are mechanisms hat restrict access to resources to the users who are speciafically allowed to access those resources
* For example, a banking application will allow a user to view transaction and make payments from their own accounts, but not the accounts of any other user.
* Horizontal privilege escalation is a bit tricky to pull of as it requires the attacker to gain access to the account credentials as well as elevating the permission

SQLI UNION Attack:

The UNION operator is used to combine the result set of two or more SELECT statement. Every SELECT statement within UNION mst have the same number of columns. The columns must also have smililar data types. The columns in every SELECT statemtnt must also be in the same order

Attack::

We can use he union keyword to retrieve data from other tables within the database. This is commonly known as SQL injection UNION attack. The UNION keyword enables you to execute one or more additional SELECT queries and append the results to the orginal query. EG: SELECT a,b FROM table1 UNION SELECT c,d FROM table2

Requirement:
How many columns are being returned from the original query. Which columns returned from the original query are of a suitable data type to hold the results from the injected query.

Determining number of Columns:

There are 2 ways to determine number of Column:
* By using ORDER BY clause
* Null Operator
The ORDER BY clauses are used frim uncrementing the specified column index untill an error occurs:

Eg:
* 'ORDER BY 1--
* 'ORDER BY 2--
* 'ORDER BY 3--

Error: The ORDER BY position number 3 is out of range of the number of items in the select.

By using Null Operator:


NULL is convertible to every common data type, so it maximizes the chance that the payload will succeed when coulmn count is correct:
EG:
* 'UNION SELECT NULL--
* 'UNION SELECT NULL,NULL--
* 'UNION SELECT NULL,NULL,NULL--
Error: All queries combined using a UNION, INTERSECT or EXCEPT operator must have to equal number oof expressions in their target lists


Blind SQL Injection:

In Blind SQLI, the HTTP response do not contain the results of the relevant SQL query or the details of any database error. Many Techniques such as UNION attacks are not effective with blind SQLI vulnerabilities. We can also call it as inferential SQLI. It is still possible to exploit blind SQLI to aaccess unauthorized data but different techniques must be used. There are 2 types Time based and Boolean Based.

It is known as content-based Blind SQLI. In this the attacker makes different SQL queries that ask the database TRUE or False questions. Then they analyze difference in responses between TRUUE and False statements.

Eg:
param = xyz' AND '1'='1 > TRUE

Time based BLind SQL Injection:
In this, an attacker sends specific SQL queries that force the database to "sleep" or pause for a specified period, thus confirming the vulnerability. It is used when the application is configured to suppress error messages but has not sufficiently mitigated the underlying SQLI vulnerability. The response time will indicate to the attacker whether the result of the query is  TRUE or FALSE.
